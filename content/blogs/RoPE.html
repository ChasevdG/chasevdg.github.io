Rotary Positional Embeddings (RoPE) have become the de facto positional encoding in modern large language models -- Llama, Mistral, DeepSeek, Gemma all use it. This post gives a self-contained review of how RoPE works, why it has the relative-position property, and how it extends to two-dimensional inputs. We cover Axial RoPE -- the standard 2D extension -- and Mixed RoPE, which replaces axis-aligned frequencies with learnable mixed frequencies to express oblique attention patterns. We also discuss p-RoPE, which zeros out the lowest-frequency dimensions to improve long-context generalization.

Background: Additive Positional Encodings

The standard transformer attention score between a query $\mathbf{q}$ and key $\mathbf{k}$ is $$a(\mathbf{q}, \mathbf{k}) = \mathbf{q}^\top \mathbf{k}.$$ Because this is symmetric in the ordering of the token sequence, attention alone is permutation-equivariant -- feeding the model a shuffled sentence gives the same attention scores as the original. Positional encodings break this symmetry by injecting positional information before the attention computation.

The classical approach, introduced with the original Transformer , is to add a position-dependent vector to each token embedding before computing queries and keys: $$\mathbf{x}'_p = \mathbf{x}_p + \mathbf{PE}(p),$$ $$\qquad \mathbf{PE}(p)_{2n} = \sin\!\left(\frac{p}{10000^{2n/d}}\right),$$ $$\quad \mathbf{PE}(p)_{2n+1} = \cos\!\left(\frac{p}{10000^{2n/d}}\right).$$ The frequency spectrum runs from fast (small $n$, high frequency) to slow (large $n$, low frequency), giving every position a unique fingerprint. Note that $\sin$ and $\cos$ can be expressed as the real and imaginary part of a complex exponential. Thus, sinusoidal positional encodings can be viewed as complex vector pairs which have a unit complex vector added to them.

Rotary Positional Embeddings

RoPE replaces the additive encoding with a multiplicative rotation. Rather than adding a fixed vector to the embedding, we rotate the query and key vectors by an angle proportional to their position. Viewing pairs of embedding dimensions as two-dimensional planes, the rotation in the $n$-th plane at position $p$ is $$R_{\theta_n}(p) = \begin{pmatrix} \cos(p\theta_n) & -\sin(p\theta_n) \\ \sin(p\theta_n) & \cos(p\theta_n) \end{pmatrix}, \qquad \theta_n = 10000^{-2n/d}.$$ Stacking these blocks gives a block-diagonal rotation matrix acting on the full $d$-dimensional space, $$R(p) := \mathrm{blkdiag}\!\left(R_{\theta_0}(p),\, R_{\theta_1}(p),\, \ldots,\, R_{\theta_{d/2-1}}(p)\right).$$ RoPE is then applied directly to the query and key inside each attention head -- not to the token embedding itself: $$\tilde{\mathbf{q}}_p = R(p)\,\mathbf{q}_p, \qquad \tilde{\mathbf{k}}_p = R(p)\,\mathbf{k}_p.$$

In the complex-number view, pairing dimensions $(2n, 2n+1)$ into a single complex number $q_n \in \mathbb{C}$, the rotation becomes simply $$\tilde{q}_n = q_n\, e^{i p \theta_n},$$ which is a compact and elegant view of the rotation pairs as a multiplication by a unit complex number.

The Relative Position Property

The key motivation for RoPE is that the attention score between a query at position $m$ and a key at position $n$ depends only on the relative offset $m - n$. This follows directly from the rotation structure. Since $R(p)$ is orthogonal, $R(p)^\top = R(-p)$, and we have $$\tilde{\mathbf{q}}_m^\top \tilde{\mathbf{k}}_n = \bigl(R(m)\mathbf{q}\bigr)^\top\bigl(R(n)\mathbf{k}\bigr) = \mathbf{q}^\top R(m)^\top R(n)\,\mathbf{k} = \mathbf{q}^\top R(n-m)\,\mathbf{k}.$$ The product of two rotation matrices $R(m)^\top R(n) = R(n-m)$ is a rotation by the difference in positions. Crucially, computing $R(m)$ and $R(n)$ independently and combining at attention time is equivalent to computing the relative rotation $R(n-m)$ directly, but the former requires only $O(N)$ computation rather than $O(N^2)$.

This also means that if both the query and key are shifted by the same constant $\delta$, the attention score is unchanged: $$\mathbf{q}^\top R((n+\delta)-(m+\delta))\,\mathbf{k} = \mathbf{q}^\top R(n-m)\,\mathbf{k}.$$ The animations on the left confirm this geometrically: translating both positions together leaves the inter-vector angle -- and therefore the attention score -- invariant.

Is RoPE Local?

A persistent piece of folklore is that RoPE imparts a built-in locality bias: attention is supposed to decay smoothly with relative distance, so that far-apart tokens naturally attend to each other less. The original RoFormer paper motivated RoPE in exactly these terms, deriving a long-term decay property -- an upper bound on the attention score that shrinks as $|m-n|$ grows. This is often repeated as a reason RoPE "just works": it is said to encode a soft prior that nearby tokens matter more.

p-RoPE

If the query and key begin aligned, then the attention will decay as suggested in . However, this is not generally the case. In fact, Barbero et al. show that for any distance there is a token that has its attention maximized at that distance.

Moreover, Barbero et al. show empirically that trained models do not use RoPE to enforce decay; instead the network exploits the spectrum in the opposite way. The high-frequency dimensions are used for genuinely position-sensitive, local matching, while the low-frequency dimensions -- the ones that rotate slowly enough to be nearly constant across the context -- are repurposed as approximately position-invariant channels for carrying semantic content and constructing robust long-range attention (for instance, the attention sinks that let a query reliably reach back to the first token). Far from being local, RoPE's slow components are precisely what enable strong attention at arbitrary distance.

These same slow components are, however, the source of RoPE's out-of-distribution failure. Barbero et al. find that the low-frequency dimensions are precisely those that develop large, coherent attention scores beyond the training context length -- the very property that makes them useful within the training window makes them harmful outside it. This tension motivates a direct intervention on the frequency spectrum.

By setting proportion of the RoPE frequencies to invariant ($\theta=0$), they found that the model generalized better in to longer contexts. For a deeper discussion, see the following blog post.

Axial RoPE (2D)

In vision transformers tokens are image patches with a two-dimensional grid position $(p_x, p_y)$. To extend RoPE to 2D, the natural first attempt is to apply a rotation for $p_x$ followed by a rotation for $p_y$: $$R(p_x, p_y) := R(p_x)\,R(p_y).$$ Since both matrices are block-diagonal with the same block structure, their product is also block-diagonal. Moreover, a 2D rotation block satisfies $R_\theta(p_x)R_\theta(p_y) = R_\theta(p_x + p_y)$, so the combined rotation reduces to $$R(p_x, p_y) = \mathrm{blkdiag}\!\left(R_{\theta_n}(p_x + p_y)\right).$$ This is simply a 1D RoPE applied to the scalar $p_x + p_y$, which collapses the 2D coordinate to a one-dimensional sum and cannot distinguish positions that share the same sum -- e.g. $(1,3)$ and $(2,2)$ become identical.

The standard fix is to split the embedding dimensions and assign the first half to $p_x$ and the second half to $p_y$: $$R_\text{2D}(p_x, p_y) := \mathrm{blkdiag}\!\Bigl(\underbrace{R_{\theta_n}(p_x)}_{n < d/4},\; \underbrace{R_{\theta_n}(p_y)}_{n \geq d/4}\Bigr).$$ The attention score then factors into independent contributions from the horizontal and vertical offsets: $$\tilde{\mathbf{q}}^\top \tilde{\mathbf{k}} = \mathbf{q}_x^\top R_{\theta}(\Delta x)\,\mathbf{k}_x \;+\; \mathbf{q}_y^\top R_{\theta}(\Delta y)\,\mathbf{k}_y,$$ where $\Delta x = p_x^{(q)} - p_x^{(k)}$ and $\Delta y = p_y^{(q)} - p_y^{(k)}$. This correctly preserves 2D relative position: shifting both patches by the same $(dx, dy)$ leaves the score unchanged.

A fundamental limitation of Axial RoPE is that its attention score factors as $\alpha_{ij} = \alpha^{(x)}_{ij} + \alpha^{(y)}_{ij}$, meaning it can only represent separable, axis-aligned patterns . To represent attention that is sensitive to oblique directions -- e.g. patches that are offset both horizontally and vertically -- the frequencies must contain components along both axes simultaneously.

Mixed RoPE

Mixed RoPE addresses this by replacing the axis-aligned fixed frequencies of Axial RoPE with a learnable frequency vector per dimension. Each embedding plane $d$ gets its own pair of frequency weights $(\omega_{dx}, \omega_{dy})$, and the 2D rotation is $$\phi_\mathbf{\omega}(z_d,\, \mathbf{p}) = R_{\mathbf{\omega}^\top \mathbf{p}} = R_{\omega_{dx}\, p_x\, +\, \omega_{dy}\, p_y}\; z_d.$$ In full block-diagonal form, $$R_\text{Mixed}(\mathbf{p}) = \mathrm{blkdiag}\!\left(R_{\omega_{0x} p_x + \omega_{0y} p_y},\; R_{\omega_{1x} p_x + \omega_{1y} p_y},\; \ldots\right).$$ Axial RoPE is the special case in which each dimension has either $\omega_{dy}=0$ (pure $x$-frequency) or $\omega_{dx}=0$ (pure $y$-frequency). Mixed RoPE lifts this restriction, allowing the learned frequencies to point in any direction in the $(p_x, p_y)$ plane.

Summary

RoPE encodes position as a block-diagonal rotation applied to queries and keys, so that attention scores depend only on the relative offset $m-n$ at $O(N)$ cost. Despite early claims of a built-in locality bias, RoPE is not inherently local: trained models repurpose the low-frequency dimensions for long-range attention rather than decay. Those same dimensions, however, develop coherent oscillations beyond the training context length, motivating p-RoPE , which zeros them out to improve out-of-distribution generalization.

In two dimensions, Axial RoPE splits the embedding dimensions between the $x$- and $y$-axes, correctly encoding 2D relative position but limited to separable, grid-aligned attention patterns. Mixed RoPE lifts this restriction with learnable per-dimension frequency vectors $(\omega_{dx}, \omega_{dy})$, allowing each plane to point in any direction and enabling richer, oblique attention patterns.
